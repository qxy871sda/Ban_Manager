# 全局黑名单管理器 (Ban Manager)
## ⚠️ 安全风险警告

> **现在该插件存在以下严重的设计缺陷与安全/稳定性风险，非专业人士请不要使用，以防数据损坏或系统崩溃。**

### ⚠️ 已知严重风险说明

该插件在运行机制上存在以下**待修复**的**安全问题**：

#### 1. 跨插件越界修改配置文件，绕过 SDK 安全机制
插件通过硬编码路径（如 `plugins/maibot-team_snowluma-adapter/config.toml`）直接读写其他适配器的配置文件，绕过了 SDK 提供的正规插件间通信机制。
* **潜在后果**：一旦目标适配器未来更新并调整了配置结构，或者用户手动修改了路径，将直接导致配置损坏。此外，这种做法越过了 SDK 声明的 `capabilities` 限制（清单中仅声明了只读能力 `config.get`，但实际执行了写操作），属于越界写入行为。

#### 2. 启动时自动合并与强制覆盖，存在数据破坏风险
在插件加载时，方法 `_initialize_ban_list_on_load` 会自动获取适配器已有名单并与本插件名单取并集，随后**直接写回**所有适配器的配置文件。
* **潜在后果**：该行为在用户尚未备份或确认配置的情况下强制执行，属于过于激进的“自动覆盖”操作，极易造成用户原始数据丢失或损坏。

#### 3. 设置项失效，可能导致非预期的意外覆盖
配置项 `adapter_type` 存在不一致行为。在 `_initialize_ban_list_on_load` 和 `_sync_to_adapters` 两个方法中，插件会向所有检测到的适配器强制写入配置，完全忽略了用户设置的 `adapter_type`。
* **潜在后果**：即使用户将 `adapter_type` 显式设为 `none`，启动时其他适配器的配置文件仍有被强行修改和覆盖的风险。
  
## 简介
本插件能为 麦麦（Bot） 在适配器内`全局屏蔽用户`名单内`添加/移除`QQ用户提供支持（工具调用）。它也支持管理员通过指令手动加入`全局屏蔽用户`，也可以在`config.toml`中自定义调用规则自主判定并屏蔽对应发言者。本插件会将对应用户QQ 自动/双向同步至 `SnowLuma`或`NapCat`配置器内`config.toml`中的`ban_user_id = []`条目，实现更底层、更高效的无感消息过滤。

## 核心功能
- **主动忽略消息**：通过调用 OneBot 适配器的 `ban_user_id` 机制，bot 会在适配器层直接丢弃黑名单内用户的消息，实现主动屏蔽（非群内禁言），此屏蔽为全局的屏蔽。
- **名单同步**：自动检测并同步 SnowLuma/NapCat 适配器的黑名单，支持在 WebUI 直接管理屏蔽名单，修改后自动实时同步。
- **以工具调用**：内置 `add_to_global_blacklist` 工具，允许 麦麦 在检测触发到用户自定义设置的规则后，自主决定将其拉入黑名单。
- **灵活的群聊过滤**：支持白名单/黑名单模式，您可以精确指定哪些群聊受此插件规则影响。

## 指令列表 (管理员可用)
*   `/拉黑 <QQ号> [原因]` / `/屏蔽 <QQ号> [原因]`：手动将指定用户加入全局屏蔽名单。
*   `/解除 <QQ号>`：将指定用户从全局屏蔽名单中移出。
*   `/屏蔽名单` / `/拉黑名单`：列出当前所有被屏蔽的 QQ 号。

## WebUI 配置说明
（**建议在启用前先备份原有适配器的 `config.toml`或`config.toml`内`ban_user_id = []`**）
**如果适配器或本插件名单有一方为空，则可能被同步清空所有`ban_user_id`内容**
- **当前适配器类型**：
    - `none`：默认项，请备份好原有名单再进行更改。
    - `snowluma` / `napcat`：插件将读取并自动更新对应适配器的配置文件。
    - `all`：插件将同步管理两个适配器的屏蔽名单。
- **发送群内通知**：开启后，当有用户被屏蔽时，bot 会在群内发送成功等通知消息。
- **修改屏蔽名单**：支持在此界面直接添加、编辑或删除 QQ 号，修改后自动同步至各适配器。
- **工具描述**：支持自定义，您可以修改发送给 麦麦 的工具描述，从而自定义拉黑的标准（如：将“谩骂”改为“骚扰”等）。

## ⚠️ 免责与注意事项
1. **屏蔽机制**：本插件是通过适配器配置文件中的 `ban_user_id` 实现的“不接收消息”，并非 QQ 群内的禁言功能，被屏蔽的用户依然可以在群内正常发言，但 bot 将完全看不见其消息。
2. **配置文件备份**：在将 `adapter_type` 修改为 `all` 或批量迁移名单前，请务必备份您的适配器 `config.toml` 文件。
3. **加载顺序**：本插件在启动时会自动合并适配器中已有的名单，确保 WebUI 能够完整显示现有屏蔽记录。
